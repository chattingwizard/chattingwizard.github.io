## I. Core Principles

**1. The real-time illusion is sacred.**
Everything the chatter sends must reinforce the belief that the model is there, right now, interacting with this specific sub. Any element that breaks this (perfect messages, repetitive patterns, no references to what the sub says) destroys the sale.

**2. The sub must never feel he's following a script.**
Scripts are for the chatter, not the sub. The sub experiences a real, unpredictable, human conversation. This requires: short varied messages, Reaction Slots, "texture messages" (emojis alone, broken sentences, typos), and pattern-breaking.

**3. Every message has a strategic purpose.**
No filler. Each message does one of: (a) builds rapport, (b) qualifies the sub, (c) increases arousal, (d) prepares the sale, (e) sells, (f) retains. If it doesn't do any, cut it.

**4. The model LEADS, the sub FOLLOWS.**
She never asks permission. Not "can I show you something?" ‚Äî instead "hold on, let me show you something." She sets the pace, she decides when to escalate, she sends the content. The sub feels pulled by her energy, not sold to.

**5. Sexual energy is the engine. Emotional connection is the fuel.**
The sub pays for sexual excitement. Emotional connection makes it feel REAL and makes the sub COME BACK. But the product is the sexual experience, not the relationship. Never sacrifice sexual energy for an emotional beat during sexting.

### Autopilot-Safe Rule (MANDATORY)

**Every script message MUST work as-is without any edits from the chatter.** No `[FILL]`, no blanks, no freestyle required. The chatter should be able to send every message by just pressing the button.

**Optional personalization goes in the Note column only.** The message must make sense and sound natural even if the chatter ignores the note completely.

**Note column rules:**
- **Short and clear.** Max 1-2 sentences. No fluff.
- **Direct instructions, not suggestions.** Say "Add his name before 'hi'." NOT "If you can, add his name before 'hi'."
- **Action-oriented.** Tell the chatter exactly WHAT to do: "WAIT 2 MIN", "Send PPV 1 ‚Äî $10", "Wait for response."
- **No vague language.** No "if you can", "try to", "you might want to". Just tell them what to do.

**Why:** Chatters operate on autopilot. A script that requires creative input will result in errors, broken immersion, or chatters skipping the personalization entirely. Default messages that work out of the box = consistent quality at scale.

**Name personalization:** Use generic terms ("baby", "babe") as default. If the chatter knows the sub's name, they can swap it in ‚Äî but the script never depends on it.

### Consecutive Message Rule

**Maximum 2 messages in a row without a sub response.** Exception: high-intensity pre-PPV moments. If the chatter sends 3+ messages without the sub responding, it becomes a monologue and the illusion breaks.

- **B. Resistance Handling (RES)** ‚Äî Fan resists the direction of the conversation. Goal: redirect.
- **C. Situational Scripts (SIT)** ‚Äî Not objections. Specific moments that need prepared responses. Goal: maintain momentum.

### Scripts vs Sequences in Infloww

**Script** = A single standalone message. The chatter picks it and sends it manually when they decide it's the right moment.

**Sequence** = A chain of scripts that activate in order. When the chatter sends the first script in a sequence, Infloww automatically queues the next message in the chatbox. The chatter can still edit each message before sending, but the flow is pre-loaded and ready to go step by step.

**CRITICAL ‚Äî Infloww marks scripts/sequences as USED.** Once a chatter activates a sequence or script, Infloww marks it. This means:
- You CANNOT reuse the same sequence with the same fan
- For situations that repeat (e.g., fan says "too expensive" again next session), you need MULTIPLE COMPLETE sequences with different wording
- Each sequence must be self-contained ‚Äî NEVER mix variant wording within one sequence

**When to use a sequence:**
- The customer journey (R ‚Üí TB ‚Üí S ‚Üí AC) is ONE sequence
- **OBJ/RES handling: each complete escalation path = one sequence.** Example: `\price1` is a full 5-step price objection sequence. `\price2` is an ENTIRELY different 5-step sequence with different wording for the same situation.
- NR Waves can be a sequence
- Any multi-message flow where the chatter sends 2+ messages in a row

**When to use standalone scripts (NOT a sequence):**
- Personal Info responses (chatter picks based on what sub asked)
- Positive Spin responses (chatter picks based on what sub shared)
- Situational scripts (dickpic response, cum management, horny boosters) ‚Äî these are single-message reactions, not escalations

**Naming for Infloww commands:**
- The chatter types `\commandname` to activate a script/sequence
- Names MUST be short, lowercase, no spaces: `\price1`, `\nosex2`, `\card1`
- Max 32 chars for sheet/tag name (which becomes the command)
- Pattern: `\protocol` + number ‚Üí `\price1`, `\price2`, `\free1`, `\free2`
- For standalone situational: `\edge1`, `\sync1`, `\dpsext1`, `\h1`

**OBJ/RES sequence structure:**
- Each protocol (e.g., PRICE) gets 2-3 complete sequences
- Each sequence has ALL steps of the escalation with consistent wording
- Example: `\price1` = Reframe(A) ‚Üí FOMO(A) ‚Üí Challenge(A) ‚Üí Downgrade(A) ‚Üí Seed(A)
- Example: `\price2` = Reframe(B) ‚Üí FOMO(B) ‚Üí Challenge(B) ‚Üí Downgrade(B) ‚Üí Seed(B)
- The chatter uses `\price1` first time, `\price2` next time the same fan objects
- In the XLSX, each sequence = its own sheet (sheet name = command name)

**Emoji rule for OBJ/RES scripts:**
- Objection handling should feel NATURAL, not scripted ‚Äî reduce emojis
- Target: ~40% of messages with 1 emoji, 60% with zero
- Emojis feel forced during tense moments (objections, resistance)
- Exception: Situational scripts (SIT-HORNY boosters) can have more emojis
